# AG Tools Development Rules

This document is the main development standard for all future AG Tools work.

## Brand Identity

**Name:** AG Tools

**Target audience:** Professional tilers and construction companies.

**Brand values:**

- Professional
- Reliable
- Industrial
- Modern
- Fast

## Colors

| Role | Value |
| --- | --- |
| Primary | `#FF6A00` |
| Black | `#0D0D0D` |
| White | `#FFFFFF` |
| Light Gray | `#F5F5F5` |
| Dark Gray | `#2B2B2B` |

Never introduce additional accent colors.

## Typography

Use Inter everywhere.

| Element | Weight |
| --- | --- |
| Headings | 700 |
| Body | 400 |
| Buttons | 600 |

## UI Principles

- Minimal
- Industrial
- Premium
- Lots of whitespace
- Large product photos
- Rounded corners: 12px
- Soft shadows
- Smooth hover animations

## Components

Every UI component must be reusable.

Examples:

- ProductCard
- CategoryCard
- BrandCard
- Hero
- Newsletter
- Benefits
- Header
- Footer
- Buttons
- Badges

## WooCommerce

- Never modify WooCommerce core.
- Use hooks whenever possible.
- Override templates only if absolutely necessary.

## Performance

- Mobile first.
- PageSpeed target: Desktop 95+ and Mobile 90+.
- Lazy load images.
- Use responsive images.
- Do not add unnecessary plugins.

## Coding Style

- Clean PHP.
- Modern CSS.
- No inline styles.
- No duplicated code.
- Organize files into components.

## Git

- Small commits.
- Meaningful commit messages.
- Never commit temporary files.

## Future Features

- AI search
- AI recommendations
- Product comparison
- Favorites
- Dynamic filters
- Brand pages
- Blog
- Marketplace support
